7. TROUBLESHOOTING COMÚN
================================================================================

Problema: "Failed to compile"
─────────────────────────────
Solución:
  1. Limpia cache: mvn clean
  2. Recompila: mvn compile
  3. Verifica Java: java -version (debe ser 21)
  4. Revisa mensajes de error en la salida

Problema: Puerto 8080 en uso
──────────────────────────────
Solución:
  Opción A: mvn spring-boot:run -Dspring-boot.run.arguments="--server.port=8081"
  Opción B: Mata proceso en puerto: netstat -ano | findstr LISTENING
            taskkill /PID <PID> /F

Problema: "Cannot find symbol" en compilación
───────────────────────────────────────────────
Solución:
  1. Verifica que el campo existe en el modelo
  2. Revisa fieldName (mayúsculas/minúsculas)
  3. Ejecuta: mvn dependency:resolve
  4. Limpia target: rm -rf target

Problema: JWT Token inválido
──────────────────────────────
Solución:
  1. Obtén nuevo token: POST /auth/login
  2. Copia completamente el token (sin comillas)
  3. En header Authorization: "Bearer " + token
  4. Verifica que email/password sean correctos

Problema: Email no enviado
──────────────────────────
Solución:
  1. Verifica application.properties: spring.mail.* configurado
  2. Prueba connectionURI: telnet smtp.gmail.com 587
  3. Verifica contractor/contractor@example.com existe
  4. Revisa logs: grep -i "email" Ez_back/logs/*

Problema: Excel file upload falla
─────────────────────────────────
Solución:
  1. Verifica que archivo sea .xlsx (no .xls)
  2. Revisa formato de columnas en Excel
  3. Verifica permisos de lectura del archivo
  4. Revisa logs para mensajes específicos
